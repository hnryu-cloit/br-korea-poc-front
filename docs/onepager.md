# BR Korea 점주 운영 지원 AI 대시보드 Onepager

## 한 줄 정의
도넛 매장 점주가 대시보드에서 생산, 주문, 매출 현황을 먼저 요약해서 보고, 필요 시 각 영역 안에서 바로 AI와 대화하며 의사결정을 할 수 있게 돕는 점주용 운영 대시보드다.

## 배경과 문제
- 현재 프로젝트는 점주용 운영 화면의 기본 골격은 갖췄지만, 점주 핵심 업무와 직접 연결되는 기능과 데모성 화면이 혼재되어 있다.
- 점주 인터뷰 기준 핵심 문제는 `지금 무엇을 생산해야 하는지`, `오늘 무엇을 얼마만큼 주문해야 하는지`, `매출이 아니라 실제 이익이 나는지`를 빠르게 판단하기 어렵다는 점이다.
- 생산관리에서는 재고 확인, 생산 필요 시점 판단, 재료 소진 대응, 품절 처리까지 여러 시스템과 수작업이 분리되어 있다.
- 주문관리에서는 주문 누락 방지와 추천 근거 신뢰성이 중요하며, 예측과 권고는 최소 범위로 제공하고 최종 의사결정은 점주가 수행하는 방식이 필요하다.
- 매출분석에서는 단순 매출 요약이 아니라 순이익, 손익분기점, 비교군 기반 실행 인사이트가 필요하다.
- 기존 점주 사용 흐름은 POS와 앱을 오가며 `매출 확인`, `주문 판단`, `재고 확인`, `알림 확인`을 처리하는 구조이며, 이번 개선은 이 익숙한 흐름을 유지한 채 판단과 실행을 더 빠르게 만드는 방향이어야 한다.
- 따라서 기존 화면 구조를 완전히 뒤엎기보다, 점주가 이미 익숙한 메뉴 체계와 업무 순서를 크게 해치지 않는 범위에서 정보 우선순위와 액션 동선을 개선해야 한다.
- 현재 프로젝트에는 점주 가치와 직접 연결되지 않는 본사 운영/감사 로그/시스템 정책 페이지가 상대적으로 큰 비중을 차지한다.

## 목표
- 점주 기준 핵심 3개 업무인 `생산`, `주문`, `매출/손익 분석`에 화면과 기능을 집중시킨다.
- 현재 구현을 바탕으로 유지할 기능과 제거/축소할 기능을 명확히 나눠 다음 개발 우선순위를 정한다.
- 프론트와 백엔드 기준으로 실제 구현 가능한 페이지 구조와 데이터 계약 방향을 정리한다.

## 핵심 사용자
- 1차 사용자: 매장 점주
- 2차 사용자: 점주 의사결정을 지원하는 운영 담당자
- 후속 사용자: 본사 관리자

## 핵심 가치
- 점주가 여러 화면과 외부 채널을 오가지 않고 한 흐름 안에서 생산, 주문, 분석 판단을 끝낼 수 있다.
- AI는 추천과 설명을 제공하지만 최종 의사결정은 점주가 수행한다.
- 추천 근거, 비교 기준, 예상 효과를 함께 제시해 본사 의도 개입이 아니라 데이터 기반 지원 도구로 인식되게 한다.
- 점주 화면은 실행 중심으로 단순화하고, 운영/감사/정책용 화면은 별도 관리 영역으로 분리한다.
- 기존 POS와 앱에서 점주가 익숙하게 써 온 업무 순서는 유지하되, `알림 → 판단 → 실행 → 기록`이 더 짧고 분명하게 이어지도록 만든다.

## 솔루션 구조
- 입력: POS/재고/생산 등록/주문 이력/날씨/행사/배달 채널 품절 상태/원가 및 비용 데이터
- 처리: 5분 단위 재고 스냅샷, 1시간 후 재고 예측, 4주 패턴 비교, 주문 추천 생성, 예외 상황 보정, 손익 및 손익분기점 계산, 카드 단위 컨텍스트 기반 자연어 질의 라우팅
- 출력: 대시보드 요약 카드, 생산 우선순위, 주문 추천 3안, 추천 근거, 알림, 순이익 분석, 실행 인사이트, 채널 품절 처리 액션

## 주요 기능
- 대시보드: 생산현황, 주문관리, 매출현황 3개 영역을 메인에서 요약 제공
- 생산: 5분 단위 재고 조회, 현재 재고, 1시간 후 예측 재고, 4주 평균 생산 패턴, 생산 필요 시점 자동 감지 및 알림, 재고 소진 1시간 전 PUSH 알림, 찬스 로스 감소 효과 산출
- 주문: 주문 누락 방지 중심 운영, 주문 마감 20분 전 알림, 과거 데이터 기반 3가지 주문 옵션 추천, 반복 루틴 자동 세팅, 조기 품절 보정 추천, 추천 근거 설명, 최종 의사결정은 점주 수행
- 매출/손익 분석: 자연어 질의, 예상 질문 10개 이상, 실행 가능한 인사이트, 순이익 및 손익분기점 분석, 동일 행사/직영점 비교, 매장 맞춤형 답변
- 컨텍스트 챗: 각 카드 우측 상단 인접 영역에서 카드 주제에 맞는 AI 질의를 바로 시작
- UI/UX: 중요한 데이터만 먼저 보여주는 요약형 홈, 점주용 핵심 액션 우선 구조

## 기존 사용 흐름 참고 원칙
- `POS 메인`, `점주 앱`, `AI 보조 화면` 등으로 이어지는 기존 사용 흐름은 참고 대상으로 보고, 점주가 익숙한 업무 순서를 크게 바꾸지 않는다.
- 기존 화면에서 점주가 자주 쓰는 `매출 확인`, `주문 확인`, `재고 확인`, `알림 확인`의 진입 구조는 유지하되, 각 화면에서 바로 다음 행동을 할 수 있도록 개선한다.
- 기존 플로우를 버리고 완전히 새로운 탐색 구조를 만드는 것이 아니라, 같은 흐름 안에서 더 적은 클릭과 더 명확한 우선순위를 제공하는 것이 목표다.
- AI는 기존 흐름을 대체하는 별도 제품이 아니라, 점주의 기존 운영 흐름 안에 자연스럽게 붙는 보조 레이어로 동작해야 한다.
- AI 진입 방식도 별도 플로팅 버튼보다 각 업무 카드에 붙은 컨텍스트형 진입이 우선이며, 사용자는 보고 있던 카드의 문맥을 유지한 채 질문할 수 있어야 한다.

## 현재 프로젝트 분석
- 현재 강점
  - `Dashboard`, `Production`, `Ordering`, `Sales` 페이지로 점주 핵심 업무의 기본 흐름은 이미 구현되어 있다.
  - 생산 페이지는 현재 재고, 1시간 후 예측, 생산 등록 액션까지 연결되어 있다.
  - 주문 페이지는 3개 추천안 비교와 점주 선택 저장 흐름이 구현되어 있다.
  - 매출 페이지는 자연어 질의형 인터페이스와 추천 질문 구조가 준비되어 있다.
- 현재 한계
  - `Sales`와 `Analytics`가 분리되어 있어 점주 관점에서 분석 경험이 중복된다.
  - `HQCoaching`, `HQInspection`, `Signals`, `Orchestration`는 점주용 POC의 핵심 가치보다 본사/데모 설명 성격이 강하다.
  - 순이익, 손익분기점, 조기 품절 보정, 날씨/행사 보정, 자동 품절 처리 등 인터뷰 핵심 요구는 아직 충분히 반영되지 않았다.
  - 요약 우선 UX보다는 정보가 분산되어 있어 점주가 무엇부터 봐야 하는지 즉시 이해하기 어렵다.
  - 기존 점주 사용 흐름을 고려하면, 새로운 기능을 추가하더라도 메뉴 구조와 액션 순서가 지나치게 낯설어지지 않도록 조정이 필요하다.

## 추가되어야 하는 내용
- 생산관리
  - 5분 단위 자동 갱신과 `마지막 갱신 시각` 명시
  - SKU별 `현재 재고 / 1시간 후 예측 / 4주 평균 1차 생산 / 4주 평균 2차 생산` 고정 노출
  - 생산 필요 시점 자동 감지 기준과 알림 트리거 명시
  - `오늘 이 제품 소진 속도가 빠릅니다` 같은 속도 기반 알림
  - 재고 소진 1시간 전 PUSH 알림과 재료-상품 연결 로직
  - `지금 생산하면 찬스 로스를 몇 % 줄일 수 있는지` 효과 추정과 산출 로직
  - 재고 기반 품절 처리 제안 또는 배달앱 품절 연동 준비
- 주문관리
  - 목적은 주문 누락 방지이며, 예측 및 권고는 최소 범위로 제공
  - 추천 3안의 기준을 더 명확히 분리
  - `전일`, `전주 동요일`, `점주 반복 패턴` 3축 고정
  - 조기 품절 품목에 대한 익일 추가 주문 제안
  - 날씨, 행사, 특수일 예외 변수 반영
  - 추천 근거를 수치와 문장으로 동시 제공
  - 최종 의사결정은 점주가 수행하도록 CTA와 문구 설계
  - 점주가 자주 수정하는 수량을 기억하는 개인화 기본값
- 매출/손익 분석
  - 매출이 아닌 `순이익` 기본 탭 추가
  - 카드 수수료, 원가, 인건비, 고정비 반영 손익 계산
  - 손익분기점 분석과 `몇 개 더 팔아야 흑자인지` 설명
  - 동일 행사 대비, 직영점 대비 비교군 분석
  - 예상 질문 10개 이상 사전 제공
  - 단순 요약이 아니라 실행 액션 중심 응답 강제
  - 매장별 운영 패턴과 최근 성과를 반영한 맞춤형 답변 제공
- 공통 UX
  - 홈에서 `지금 해야 할 일 3개` 우선 노출
  - 생산현황, 주문관리, 매출현황을 홈 카드로 요약하고 상세 페이지로 연결
  - 중요 지표만 보여주는 요약 카드와 상세 페이지 분리
  - 알림 진입 시 해당 카드/섹션 자동 포커스
  - 기존 점주 흐름에서 자주 쓰는 메뉴명과 업무 순서를 최대한 유지
  - 새 기능은 기존 플로우를 끊지 않는 보조 레이어로 삽입
  - `참고 정보`와 `즉시 액션 필요 정보`의 시각적 우선순위 분리
  - 플로팅 챗 대신 카드 우측 상단 인접 영역에서 여는 컨텍스트 챗 우선
  - 생산 카드에서는 생산 관련 질의, 주문 카드에서는 주문 관련 질의, 매출 카드에서는 매출/손익 관련 질의만 우선 제안
  - 채팅은 보조 수단으로 두고, 생산 등록/주문 확정/품절 처리 같은 핵심 액션은 버튼 기반으로 바로 수행

## 제거하거나 축소해도 되는 내용
- 점주용 1차 릴리즈 기준에서는 `매출 조회(Analytics)` 페이지를 `매출/손익 분석(Sales)`에 통합하는 것이 맞다.
- `시스템 현황(Orchestration)`은 점주 메인 네비게이션에서 제거해도 된다.
- `매출 시그널(Signals)`은 본사 모니터링 성격이 강하므로 점주용 서비스에서는 제외해도 된다.
- `주문 코칭(HQCoaching)`과 `생산 점검(HQInspection)`은 점주용 POC 범위에서는 숨기고 관리자 전용 후속 범위로 넘기는 것이 맞다.
- 점주 화면에서 감사 로그, 정책 로그, SQL/API 우선 처리율 같은 내부 운영 지표는 노출 우선순위가 낮다.
- 매출 도메인에서 `매출 현황`과 `매출 조회`를 나눠 두는 구조는 유지 가치가 낮다.
- `AIoT 계획 화면`처럼 준비 중 설명 성격이 강한 화면은 점주 메인 흐름에서 제외해도 된다.

## 권장 페이지 구조
- 홈 `/`
  - 오늘 해야 할 일 요약
  - 생산현황 카드 요약
  - 주문관리 카드 요약
  - 매출현황 카드 요약
  - 생산 위험 SKU 수
  - 주문 마감 잔여 시간
  - 오늘 순이익 추정
  - 긴급 알림과 즉시 실행 버튼
  - 각 카드 우측 상단 인접 영역의 컨텍스트 챗 진입
  - 바로가기: 생산, 주문, 손익 분석
- 생산관리 `/production`
  - SKU별 현재 재고, 1시간 후 예측 재고
  - 4주 평균 생산 패턴
  - 생산 필요 시점 자동 감지 및 알림
  - 생산 등록 액션
  - 찬스 로스 감소 효과
  - 품절 처리 제안
- 주문관리 `/ordering`
  - 주문 마감 카운트다운
  - 추천 3안 비교
  - 추천 근거와 예외 변수 설명
  - 예측/권고 최소화 원칙과 점주 최종 결정 UI
  - 점주 반복 패턴 기반 기본값
  - 최종 선택 및 저장
- 손익분석 `/sales`
  - 자연어 질의
  - 예상 질문 리스트
  - 오늘/주간 순이익 요약
  - 손익분기점 분석
  - 동일 행사/직영점 비교
  - 실행 인사이트
  - 매장 맞춤형 답변
- 알림함 `/alerts` 또는 홈 내 통합 섹션
  - 생산 긴급 알림
  - 주문 마감 알림
  - 재료 소진 알림
  - 알림 클릭 시 관련 페이지 해당 영역으로 이동
- 관리자 전용 후속 영역 `/admin/*`
  - 본사 주문 코칭
  - 생산 점검
  - 전국 시그널

- 정책/감사 로그 화면의 사용자용 상세 노출
- 전국 단위 시그널 모니터링의 점주용 제공

## 운영/정책 고려사항
- 점주 화면에서는 추천 근거를 반드시 노출하고, 최종 확정은 점주 액션으로 남긴다.
- 주문 Agent는 주문 누락 방지를 보조하되 예측과 권고를 과도하게 확대하지 않고, 점주의 판단을 보조하는 범위로 제한한다.
- 주문/생산 추천 결과에는 근거 데이터와 비교 기준을 함께 저장해 사후 검증 가능해야 한다.
- 손익 데이터는 역할 기반 권한 제어가 필요하다.
- 자동 품절 처리 연동 시 채널별 반영 이력과 실패 로그를 남겨야 한다.
- 자연어 질의 응답에서는 근거 데이터 출처, 비교군 기준, 계산 기준일을 명시해야 하며, 매장별 운영 맥락을 반영한 답변이어야 한다.
- 익숙한 운영 흐름을 크게 바꾸는 UI 변경은 도입 저항이 생길 수 있으므로, 기존 사용 패턴과 용어를 최대한 유지해야 한다.
- 카드 컨텍스트 챗은 현재 카드의 도메인 문맥을 기본값으로 가지되, 사용자가 다른 도메인으로 혼선 없이 이동하도록 질의 범위를 명확히 안내해야 한다.

## 성공 지표
- 점주가 홈에서 10초 내 오늘의 핵심 액션 3개를 파악할 수 있다.
- 홈 카드에서 컨텍스트 챗을 열고 첫 질문을 입력하기까지의 시간이 짧다.
- 주문 화면에서 추천안 중 하나를 선택하거나 수정해 저장하는 시간이 기존 대비 줄어든다.
- 생산 알림 이후 실제 생산 등록 전환율이 측정 가능하다.
- 조기 품절 SKU 수와 찬스 로스 추정치가 감소한다.
- 매출 질의 중 `순이익`, `손익분기점`, `왜 줄었는지` 유형 질문에 대해 실행 액션 포함 응답 비율이 높아진다.
- 기존 점주 흐름을 크게 바꾸지 않고도 핵심 업무 처리 시간이 줄어든다.
